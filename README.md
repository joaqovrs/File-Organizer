# File Organizer

A minimalist desktop application that organizes the files in a folder by automatically grouping them into subfolders based on their extension.

Select a folder, and with a single click, all its files are moved to subfolders named according to their type: .pdf files go to pdf/, .png files to png/, .docx files to docx/, and so on.

## ✨ Features

- **One-button operation**: Choose the folder, and the app does the rest.

- **Grouping by extension**: Automatically creates a subfolder for each file type (lowercase).

- **Files without extensions**: Grouped in an "other/" folder.

- **Non-destructive operation**: If a file with the same name already exists in the destination folder, it skips it and notifies you instead of overwriting it.

- **Clear Summary**: Upon completion, it shows how many files were moved, which folders were created, and any warnings.

- **Native and Lightweight**: Built with Tauri (small binary, no Electron).

> **Note:** The organization is **not recursive**. Only files directly in the chosen folder are moved; existing subfolders are ignored.

## 🚀 Usage

1. Open **File Organizer**.

2. Click on **Select Folder**.

3. Choose the folder you want to organize from the drop-down menu.

4. Done: The files are grouped into subfolders by extension, and you'll see a summary of the operation.

## 🛠️ Compile from Source (Developers)

### Prerequisites

- [Node.js](https://nodejs.org/) (includes npm)
- [Rust and Cargo](https://www.rust-lang.org/tools/install)
- Tauri's [system dependencies](https://tauri.app/start/prerequisites/) for your operating system

### Steps

```bash
# 1. Clone the repository
git clone https://github.com/joaqovrs/File-Organizer.git
cd File-Organizer

# 2. Install the frontend dependencies
npm install

# 3. Run in development mode (Vite + Tauri)
npm run tauri dev

# 4. Or generate the production installer/binary
npm run tauri build
```

The installer generated by `npm run tauri build` results in:

```
src-tauri/target/release/bundle/
```

### Other Useful Commands

```bash
# Check frontend types
npx tsc --noEmit

# Compile only the Rust backend (faster than a full build)
cd src-tauri && cargo check
```

## ⚙️ How it works

The frontend invokes the command Rust uses `organize_files`, passing the selected path. The backend iterates through the direct files in the directory, determines the extension of each, creates the corresponding subfolder if it doesn't exist, and moves the file within it. The move is performed using `fs::rename`, which is atomic when the source and destination are on the same file system. Upon completion, it returns a summary (files moved, folders created, and warnings) that the UI displays to the user.
